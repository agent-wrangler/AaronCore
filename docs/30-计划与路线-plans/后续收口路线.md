# 后续收口路线

最后更新：2026-03-13

## 第一阶段

- 保持 `agent_final.py` 作为唯一运行入口
- 保持 `memory_db/` 作为唯一主状态目录
- 保持 `output.html` 作为唯一主页面

## 第二阶段

- 把 `agent_final.py` 拆成 `api/`、`services/`、`storage/`
- 把聊天编排、路由决策、状态读写从入口文件中拆开
- 给当前接口补最基本的依赖说明和启动说明

## 第三阶段

- 把 `output.html` 拆成独立静态资源
- 降低前端对字符串拼接和 `localStorage` 的依赖
- 让桌面启动流程能明确提示“后端未启动 / 启动失败”

## 第四阶段

- 清理仓库里剩余的历史噪音文件
- 视情况补 `requirements.txt` 或 `pyproject.toml`
- 再决定是否继续做更完整的模块化重构
