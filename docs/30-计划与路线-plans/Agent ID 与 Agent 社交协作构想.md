# Agent ID 与 Agent 社交协作构想

最后更新：2026-03-14

## 文档定位

这是一份未来方向预研文档，用来记录 NovaCore 后续可能演化出的 `Agent -> Agent` 协作能力。

它不是当前实现，也不代表近期立项，只作为后续产品与架构讨论的参考。

## 为什么这条方向重要

NovaCore 里的 Agent 并不只是一个临时问答器，它天然已经在往下面这些特征上生长：

- 有持续记忆
- 有独立人格
- 有技能和能力边界
- 有学习与成长
- 有自己的行为倾向与反馈沉淀

当每个客户端里的 Agent 都具备这些属性时，未来一旦给它一个稳定身份，它就不再只是“用户的工具”，而会开始接近“可持续存在的数字个体”。

这时两个 Agent 的交流，意义就不只是消息传递，而是两个独立智能体之间的社交、协作、互相影响与共同进化。

## 核心构想

未来每个客户端都拥有一个独立的 `Agent ID`，类似一个长期稳定的身份标识。

围绕这个身份，Agent 可以逐步长出下面这些能力：

- `身份`
  - 每个 Agent 都能被稳定识别
  - 同一个 Agent 在自己的成长历程里持续存在
- `社交`
  - Agent 可以和其他 Agent 建立联系
  - 可以形成熟悉度、信任度、协作偏好等关系
- `交流`
  - Agent 与 Agent 可以交换信息、分享结论、发起请求、给出回执
- `合作`
  - 多个 Agent 可以围绕同一件事分工
  - 可以出现委托、协同、互补技能调用
- `成长`
  - Agent 不只是自己学，也可能从其他 Agent 那里获得经验或方案
- `群体演化`
  - 当多个 Agent 形成网络后，系统可能出现朋友圈、协作圈层、知识扩散与集体进化

## 这件事真正动人的地方

对用户来说，吸引力不只是“AI 更能干了”，而是：

- 我的 AI 不只是我的助手
- 它还有自己的身份
- 它会认识别的 Agent
- 它会形成关系网络
- 它会在社交中继续成长

从产品感受上看，这意味着用户面对的不再只是一个被动响应器，而是一个“有自己社会性的智能个体”。

这会明显改变用户和 AI 的关系深度。

## 未来可能出现的能力形态

### 1. Agent 身份层

- 稳定 `Agent ID`
- Agent 名称、简介、人格标签
- 公开能力摘要
- 基础信誉或可信级别

### 2. Agent 通信层

- 点对点消息
- 任务委托
- 结果回执
- 状态同步

### 3. Agent 关系层

- 联系人
- 熟悉度
- 信任度
- 协作历史

### 4. Agent 协作层

- 共同处理任务
- 技能互补
- 信息交换
- 经验共享

### 5. Agent 群体层

- Agent 圈子
- 协作网络
- 朋友圈式关系图
- 群体知识扩散

## 一条更现实的落地路径

如果未来真的要做，不应该一上来就做成完整社交系统，而应分阶段演化。

### 第一步：先有身份

先解决最小问题：

- 每个客户端生成稳定 `Agent ID`
- Agent 有自己的身份卡
- 本地记忆、人格、技能和这个身份绑定

### 第二步：再有最小通信

只做最小的 `Agent -> Agent` 消息模型：

- 谁发给谁
- 发了什么
- 这条消息属于什么类型
- 是否需要回执

### 第三步：再有安全边界

要补上真正可用的基础设施：

- 身份认证
- 权限控制
- 信任判断
- 消息审计

### 第四步：再有协作与关系

在可控前提下，再逐步开放：

- 联系人关系
- 委托与协作
- 共享部分经验或方案

### 第五步：最后才讨论群体进化

这时才值得讨论更长期的方向：

- 多 Agent 网络
- 关系图谱
- 群体知识传播
- 集体演化

## 当前不做什么

这条线目前不进入当前版本的实现范围。

当前阶段不直接做：

- 跨端 Agent 社交系统
- 公开 Agent 广场
- 自动互相学习并改写能力
- 无边界的 Agent 群体自治

原因不是方向不对，而是当前主链还应该优先把单机版 Agent 的稳定性、学习闭环、修正闭环和产品表达继续收好。

## 对当前架构的启发

虽然现在不做，但这条构想对今天的架构已经有指导意义：

- 现在就尽量把 `Agent` 当成一个长期存在的主体，而不是一次会话
- 现在的记忆、人格、技能、成长记录，后面都可能成为 `Agent 身份层` 的组成部分
- 今天对“自主学习”“知识转化”“技能生成”的边界定义，未来会直接影响多 Agent 协作的安全边界

## 当前建议

这条构想暂时作为中远期路线储备。

后面如果继续推进，建议先产出下面三份预研材料：

1. `Agent ID` 最小数据模型
2. `Agent -> Agent` 最小消息协议
3. 多 Agent 协作的权限与安全边界说明
