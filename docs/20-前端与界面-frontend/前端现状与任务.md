# 前端现状与任务
最后更新：2026-03-16 02:42

## 当前前端架构

前端已完成模块化拆分，不再是单文件 HTML。当前结构：

```
output.html              (78 行，纯 HTML 骨架 + script/link 引用)
static/css/main.css      (1091 行，全部样式)
static/js/utils.js       (55 行，公共工具函数：escapeHtml、cleanInlineText)
static/js/awareness.js   (184 行，感知球模块)
static/js/chat.js        (342 行，聊天/发送/思考气泡)
static/js/memory.js      (142 行，记忆页)
static/js/settings.js    (554 行，设置页 5 个区块)
static/js/docs.js        (225 行，文档页)
static/js/app.js         (320 行，标签路由/初始化/侧边栏/主题切换)
```

- 页面由 `agent_final.py` 的 `GET /` 返回 `output.html`
- 静态资源由 FastAPI StaticFiles 挂载 `/static` 路径提供
- JS 加载顺序：utils → awareness → chat → memory → settings → docs → app
- 当前主栏目：`聊天`、`技能`、`消耗`、`记忆`、`设置`、`文档`

## 当前已经稳定的部分

- 聊天主链已经重新接回主后端
- 技能页、记忆页、设置页、文档页都已经恢复可用
- 设置页按钮点击问题已收口到事件委托处理，不再依赖散落的内联 `onclick`
- 学习强度的基础 / 进阶 / 深度三档，已经和后端真实配置打通
- 思考气泡已改为使用后端 `data.trace` 数据，不再做客户端关键词匹配

## 设置页当前结构

设置页当前对外分成 5 个区块：

1. `自主学习`
2. `学习强度`
3. `知识转化`
4. `修复提案审核`
5. `开发者细调`

其中：

- 前 4 个是用户可直接理解的主区块。
- `开发者细调` 是底层策略区，不是第二套审批页。

## 设置页当前交互要点

### 自主学习

- 使用 `先暂停学习 / 重新开始学习` 控制总开关。

### 学习强度

- 使用 `基础 / 进阶 / 深度` 三张卡片切换预设。
- 当前配置与任一预设不完全一致时，显示为 `当前是自定义组合`。

### 知识转化

- 统一展示最近结果态：
  - `积累经验`
  - `生成方案`
  - `自动修改`
  - `解锁新技能`

### 修复提案审核

- 当前唯一显式审批入口。
- 当前保留的用户动作：
  - `刷新提案`
  - `展开详情 / 收起详情`
  - `生成改动预览 / 重新生成预览`
  - `批准并应用`

### 开发者细调

- 当前用于控制底层开关与参数：
  - `整理修复提案`
  - `先跑最小验证`
  - `低风险自动落地`
  - `中高风险改动怎么处理`
  - 以及若干搜索、摘要、超时参数

## 当前前端改进方向

- `settings.js`（554 行）仍然偏大，可考虑按区块进一步拆分
- 仍然大量使用 `innerHTML` 拼接，后续可考虑模板化
- 前端零测试覆盖，关键交互缺少回归保障
- 仍有部分状态保存在前端内存和 `localStorage` 中
