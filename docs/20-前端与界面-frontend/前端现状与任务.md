# 前端现状与任务

最后更新：2026-03-13

## 当前前端事实

- 当前唯一主页面是仓库根目录的 `output.html`
- 当前页面由 `agent_final.py` 的 `GET /` 直接返回
- 页面仍然是单文件 HTML，样式、脚本和结构写在一起
- 当前主栏目已经恢复到聊天、技能、数据看板、记忆、设置

## 当前优点

- 页面已经有明确的工作台感，不再只是单纯聊天框
- 聊天主流程、技能页、记忆页、统计页已经重新接回主后端
- 主题切换、侧栏宽度、本地聊天缓存这些体验功能都还在

## 当前问题

- `output.html` 仍然偏大，结构和脚本难维护
- 前端大量依赖 `innerHTML` 拼接，后续容易继续膨胀
- 页面状态有一部分仍然依赖 `localStorage`
- 当前没有拆分成独立的 CSS / JS 文件
- 设置页当前已经接入自主学习配置，可直接调整 `L8` 总开关、联网补学、知识落库、负反馈纠偏补学、自动生成技能，以及补学阈值
- 侧边栏当前已新增“文档”栏，页面内可直接查看当前 README、总览、架构、前端、计划文档

## 建议下一步

- 先把 `output.html` 拆成 `styles.css` 和 `app.js`
- 再把五个主栏目拆成独立渲染函数或模块
- 最后再考虑把页面迁到更明确的前端工程结构

## 2026-03-13 记忆页当前规则

- 数据来源统一使用后端 `/memory` 返回的 `events` 与 `counts`
- `L1` 记忆粒子只参与顶部概览卡片统计，不在下方时间线渲染，避免事件流被高频聊天记录淹没
- 记忆页筛选标签当前只保留：`全部 / L3 / L4 / L5 / L6 / L7 / L8`
- `L4` 标签名称统一为“人格图谱”，事件文案采用自然语言摘要，口径向 `L5 / L6 / L8` 的结果态表达对齐
- 成长经验当前采用分层加权：
  - `L1 = min(count, 150) * 1`
  - `L3 = count * 4`
  - `L4 = count * 6`
  - `L5 = count * 18`
  - `L6 = count * 10`
  - `L7 = count * 8`
  - `L8 = count * 24`
- 成长等级当前采用递增曲线：`Lv.1 -> Lv.2` 需要 `100 EXP`，之后每一级所需经验增加 `20 EXP`
- 顶部成长卡片展示三项核心信息：总经验、本级进度、距离下一级所差经验
