# NovaCore 自动学习改造方案 v1

## 目标
把自动学习从“会乱学、乱回、不会沉淀规则”，改成“会分类、会分级、会规则化、会隔离回复”的成长链。

---

## 当前四大核心缺陷

### 1. 学到的东西没分流
用户反馈没有先判断类别，技能问题、LLM 规则问题、偏好问题、执行策略问题被混在一起。

### 2. 学到的东西没分级
系统不知道哪些反馈只影响这一次，哪些应该影响本会话、短期规则、长期规则、L5/L6。

### 3. 学到的东西没规则化
学到的不是规则，而是一段内容或一坨文本，导致无法复用，不能真正成长。

### 4. 学到的东西没边界
内部学习材料、对用户回复、规则沉淀、技能更新之间没有隔离，导致学习结果直接污染用户回复。

---

## 改造原则

### 原则 1：先分流，再学习
收到反馈后，先判断它属于哪一类，再进入对应链路。

### 原则 2：先分级，再回流
不是所有学习都能直接影响技能和回复，要先判断影响范围。

### 原则 3：学习结果必须规则化
不能只存内容，必须抽象成“以后该怎么做”。

### 原则 4：内部沉淀和外部回复必须隔离
学习材料不能直接作为用户回复输出。

---

## 最小自动学习操作系统（第一版）

### 模块 1：反馈分类器
输入：用户负反馈 / 修正反馈
输出：反馈类别

第一批类别：
- 技能路由错误
- LLM 规则错误
- 用户偏好反馈
- 执行策略错误
- 记忆/身份错误

示例：
- “这不是笑话” → LLM 规则错误
- “太短了” → LLM 规则错误（长度）
- “不该查这个” → 技能路由错误
- “你还会什么啊”后回空话 → 执行策略错误

### 模块 2：影响分级器
输入：反馈类别 + 反馈内容
输出：影响层级

第一版分级：
- once：只影响这一次重试
- session：影响当前会话
- short_term：短期规则
- long_term：长期规则

### 模块 3：规则沉淀器
输入：分类后的反馈
输出：规则项

规则项结构建议：
```json
{
  "id": "rule_20260312_xxx",
  "type": "llm_rule | skill_route | user_pref | execution_policy | memory_rule",
  "scene": "joke | story | weather | chat | identity",
  "problem": "output_not_matching_intent",
  "fix": "humor_request_should_use_llm_generation",
  "level": "session | short_term | long_term",
  "source": "user_feedback",
  "created_at": "ISO时间",
  "enabled": true
}
```

### 模块 4：回复隔离器
规则：
- 学习结果不能直接作为用户回复
- 回复必须由回复链单独生成
- 学习链只负责沉淀，不负责直接对外输出学习材料

---

## 第一阶段落地范围（低风险）
1. 新增 feedback_rules.json 作为规则沉淀入口
2. 新增 feedback_classifier.py 作为分类器骨架
3. 聊天主链在负反馈时：
   - 先分类
   - 先记录规则项
   - 再返回自然回复
4. 暂不触碰 L7/L8 → L5/L6 深回流，只做隔离和分流

---

## 预期收益
- 保住自动学习能力
- 阻止学习内容污染用户回复
- 让 L7/L8 真正开始承载“反馈分类”和“规则沉淀”
- 为后续接回 L5/L6 打下干净基础
