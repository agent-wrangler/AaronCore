# 总体架构说明

最后更新：2026-03-13

## 一句话

NovaCore 当前是一个“桌面壳 + 单体后端 + 单页前端 + 文件态记忆”的本地桌面 AI 应用。

当前有效主链：

```text
desktop.py
  -> agent_final.py
  -> http://localhost:8090/
  -> output.html
```

## 当前模块分层

### 1. 入口层

- `desktop.py`
  - 负责启动本地服务并打开桌面窗口
- `agent_final.py`
  - 当前唯一后端主入口
  - 负责 API、页面托管、上下文组装、路由编排、记忆时间线、文档接口

### 2. 前端层

- `output.html`
  - 当前唯一前端页面
  - 负责聊天、技能、统计、记忆、设置、文档六个侧边栏页签
  - 当前仍是单文件 HTML，样式和脚本尚未拆分

### 3. 表达层

- `brain/`
  - 负责人格风格、LLM 调用、本地 fallback、上下文表达
  - 聊天和技能结果最终都要经过这一层来统一口吻

### 4. 路由与执行层

- `core/router.py`
  - 当前规则优先的路由入口
- `core/executor.py`
  - 技能执行分发
- `core/skills/`
  - 当前技能目录
  - 已接入的技能包括天气、故事、代码、绘图、股票等

### 5. 记忆与状态层

- `memory_db/`
  - 当前统一状态目录
  - 这是“当前真实状态源”
- `memory/`
  - 当前主要是兼容读写封装
  - 不再作为新的状态设计来源

## 当前状态文件职责

### 对话与统计

- `memory_db/msg_history.json`
  - 结构化聊天历史
- `memory_db/stats.json`
  - 统计信息

### 记忆层

- `memory_db/long_term.json`
  - `L3` 记忆结晶
  - 现在只保留长期事实、规则、学习沉淀、里程碑等内容
- `memory_db/persona.json`
  - `L4` 人格图谱真源
- `memory_db/knowledge.json`
  - `L5` 技能矩阵/知识项
- `memory_db/evolution.json`
  - `L6` 技能执行与使用增长
- `memory_db/feedback_rules.json`
  - `L7` 负反馈规则
- `memory_db/knowledge_base.json`
  - `L8` 自主学习与补学沉淀
- `memory_db/autolearn_config.json`
  - `L8` 权限与触发阈值配置

## 当前聊天链路

### 普通聊天

1. 前端发 `POST /chat`
2. `agent_final.py` 组装 `L1/L3/L4/L5/L8 + 最近对话上下文`
3. 规则路由优先判断是否要走技能
4. 普通聊天走 `unified_chat_reply`
5. `brain.think()` 负责人格化表达
6. 回复后写历史、统计，并按条件后台触发 `L8` 补学

### 技能请求

1. 路由命中技能
2. `core/executor.py` 调具体技能
3. 技能结果回到 `agent_final.py`
4. 再交给 `brain` 做人格融合
5. 使用痕迹写入 `L6`

### 负反馈链

1. 用户出现明显负反馈
2. 写入 `L7 feedback_rules.json`
3. 后台触发 `L7 -> L8` 定向纠偏补学
4. 先沉淀内部纠偏笔记，必要时再联网补学
