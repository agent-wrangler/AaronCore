# 架构收口说明

更新时间：2026-03-13

先看总览请优先打开：`总体架构说明.md`

## 当前架构形态

NovaCore 当前是单仓、单体后端、单页前端的桌面应用：

- `desktop.py` 负责启动本地服务并打开桌面窗口
- `agent_final.py` 负责 API、页面托管、上下文组装、路由和编排
- `output.html` 是当前单页前端
- `brain/` 负责表达层和 LLM 兜底
- `core/` 负责规则路由、技能执行和技能注册
- `memory_db/` 负责统一持久化

## 这次收口采用的原则

### 1. 入口只认一条

- 运行入口只认 `agent_final.py`
- 其他 `agent*.py` 文件已移到 `archive/legacy-entrypoints/`

### 2. 规则优先，LLM 兜底

- 技能路由先走 `core/router.py`
- 只有规则层不能明确收敛时，才让 LLM 路由补判断
- 这样可以减少一次不稳定的外部调用

### 3. 状态写入统一到 `memory_db/`

- `msg_history.json`
- `stats.json`
- `persona.json`
- `knowledge*.json`
- `evolution.json`

`memory/` 目录只保留兼容读取，不再作为当前主写入目标。

### 4. 文档分成“当前”和“历史”

- `docs/00-总览-overview/`、`docs/10-架构-architecture/`、`docs/20-前端与界面-frontend/`、`docs/30-计划与路线-plans/` 只写当前事实
- `docs/99-归档-archive/` 只放历史文档
- `archive/` 目录只放历史代码、历史页面和显式备份

## 下一步仍建议继续做的事

- 把 `agent_final.py` 拆成 `api/`、`services/`、`storage/`
- 把 `output.html` 拆成独立的 CSS / JS 文件
- 去掉 `desktop.py` 中硬编码的 Python 路径
- 补 `requirements.txt` 或 `pyproject.toml`
