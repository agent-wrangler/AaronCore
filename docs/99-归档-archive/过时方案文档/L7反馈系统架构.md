# L7 反馈系统架构

> 最后更新：2026-03-17

## 一、定位

L7 是 NovaCore 的**经验沉淀层**。用户说"不对""错了""你理解错了"时，L7 记录反馈、凝结经验、生成路由约束、必要时触发代码修复。

与其他层的关系：
- L2 检测到 `correction` 类型 → 推送给 L7
- L7 自己也独立检测负反馈（`_is_negative_feedback`）
- L7 不往 L8 存东西（L8 只存纯知识）
- L7 可触发 self-repair（代码修复提案）

## 二、三级处理体系

| 级别 | 名称 | 适用类型 | 手段 | 置信度要求 |
|---|---|---|---|---|
| L1 | prompt 提醒 | 内容生成 / 交互风格 | 注入 prompt "L7经验教训" | 任意 |
| L2 | 路由约束 | 意图理解 | 动态 anti_keywords 阻断技能 | >= 0.6 |
| L3 | 代码修复 | 路由调度 / 系统故障 | self-repair 提案 | >= 0.8 |

## 三、完整链路

```
用户说"不对/你理解错了/又来了烦死了"
  │
  ├─ _is_negative_feedback()
  │   调用 classify_stage() + classify_tone()
  │   stage=correct 或 tone in (correct, complaint) → 确认是真负反馈
  │
  ├─ record_feedback_rule()
  │   ├─ classify_feedback() → 分类到 4 个板块
  │   ├─ _condense_fix() → LLM 凝结成一句可执行规则
  │   ├─ _extract_routing_constraint() → 提取路由约束（规则优先/LLM兜底）
  │   └─ 存入 feedback_rules.json（含 constraint + hit_count + fail_count）
  │
  ├─ _check_and_escalate_failed_rules()
  │   检查最近命中但仍失败的旧规则
  │   fail >= 2 → L1 升级到 L2（补生成约束）
  │   fail >= 4 → L2 升级到 L3（触发 self-repair）
  │
  └─ 按 category 分流
      ├─ 路由调度 + confidence >= 0.8 → 触发 self-repair
      └─ 其他 → 留在 L7 等下次消费
```

## 四、消费路径

### 4.1 prompt 注入（L1）

```
用户下次提问 → search_relevant_rules(msg) → 匹配相关规则
  → format_l7_context() → 注入 prompt "L7经验教训"
  → LLM 生成回复时参考
```

### 4.2 路由约束（L2）

```
用户下次提问 → _score_text() 技能评分
  → get_active_constraints()（60s 缓存）
  → match_constraint(text, constraint)（fnmatch + keywords）
  → block_skill: 技能完全跳过
  → suppress_skill: skill_score *= 0.5
```

### 4.3 self-repair（L3）

```
触发 → 后台生成提案 → 定位候选文件 → LLM 生成补丁
  → ast.parse/json.loads 语法预检
  → 备份原文件 → 写入 → 跑测试
  → 测试失败 → 自动回滚
  → 失败可重试 1 次
  → 设置页审核（进阶档需人工批准）
```

## 五、约束数据结构

```json
{
  "id": "rule_20260317_143022_123456",
  "category": "意图理解",
  "fix": "当用户提到城市时不要默认触发天气查询",
  "constraint": {
    "type": "block_skill",
    "skill": "weather",
    "patterns": ["我在*"],
    "keywords": [],
    "confidence": 0.9,
    "ttl_days": 30
  },
  "hit_count": 0,
  "fail_count": 0,
  "last_hit_at": null
}
```

## 六、涉及文件

| 文件 | 职责 |
|---|---|
| `core/feedback_loop.py` | 入口：触发检测 + 分流 + 效果追踪升级 |
| `core/feedback_classifier.py` | 分类 + LLM凝结 + 约束提取 + 检索 + 格式化 |
| `core/rule_runtime.py` | 约束缓存 + 加载 + 匹配 |
| `core/router.py` | `_score_text()` 消费约束（block/suppress） |
| `core/reply_formatter.py` | prompt 注入 "L7经验教训" |
| `core/self_repair.py` | 代码修复执行层 |
| `agent_final.py` | 初始化 + bundle 组装 + 后台任务调度 |
| `static/js/settings.js` | 设置页（学习强度 + 修复提案审核） |
| `memory_db/feedback_rules.json` | 规则存储（最多 200 条） |

## 七、安全边界

- 置信度 < 0.6 → 不生成路由约束，只走 prompt 提醒
- 置信度 < 0.8 → 不触发 self-repair
- self-repair 自动落地只允许改 `core/skills/` 下的文件
- 改核心文件（router.py 等）必须人工审核
- 补丁写入前做语法预检（.py → ast.parse，.json → json.loads）
- 写入前备份，测试失败自动回滚
- 失败最多重试 1 次，之后停在设置页等人工处理
