# NovaCore 系统说明

## 一、系统架构

NovaCore 是一个独立运行的本地AI智能体，依赖于OpenClaw的API

- **端口**: 8090
- **位置**: C:\Users\36459\NovaCore
- **模型**: MiniMax-M2.5（通过OpenClaw的API）
- **架构**: 8层AI Brain

## 二、8层Brain架构（完整版 2026-03-09）

```
                    ← 用户反馈"不对" ←
                          ↓
用户问题 → L1 → L2 → L3 → L4 → L5 → L6 → 回答
                    ↓              ↓
                   L7           (技能失败)
                    ↓              ↓
                   L8 ←←←←←←←←←←←
                    ↓
              搜索学习
                    ↓
              ┌───────┴───────┐
              ↓               ↓
         知识库           生成技能
              ↓               ↓
             L8 ─────────→ L5/L6
```

### 各层职责（2026-03-10 确立）

| 层级 | 名称 | 职责 | 存储 |
|------|------|------|------|
| L1 | 记忆粒子 | 对话历史 | msg_history.json |
| L2 | 记忆提炼 | 短期记忆（最近对话提炼） | msg_history.json |
| L3 | 记忆结晶 | 长期记忆（重要信息沉淀） | long_term.json |
| L4 | 人格图谱 | 用户+AI人格信息 | persona.json |
| L5 | 技能矩阵 | Nova"学会"了哪些技能（天气查询、画图等） | knowledge.json |
| L6 | 技能执行 | 执行了多少次技能 | evolution.json |
| L7 | 经验沉淀 | 反思（每次回答后自检） | evolution.json |
| L8 | 能力进化 | Nova"学到"了什么知识（自动搜索保存） | knowledge_base.json |

### 回流机制

1. **回答后回流**：用户反馈"不对" → L7检测 → L8搜索 → 知识/技能
2. **技能失败回流**：技能执行失败 → L7检测 → L8找备用方案 → 更新L5
3. **L8产出回流**：学到知识 → 存knowledge_base；生成技能 → 加到L5/L6

## 三、技能系统（L5+L6）

### L5：技能矩阵（配置）
- 位置：`memory_db/knowledge.json`
- 存储技能配置：trigger触发器、核心技能、辅助技能

### L6：技能执行（代码）
- 位置：`core/skills/` + `core/executor.py`
- 存储可执行函数：天气查询、讲故事、画图等

## 四、L8 自动学习（2026-03-09 新增）

### 功能
- 遇到不会的问题 → 自动 Bing 搜索 → 保存到 knowledge_base.json（纯知识）

### 智能生命体愿景（2026-03-09 思考）
- **目标**：无所不知、无所不能
- **核心**：问出问题的都是人类用户，既然问了就需要实现
- **学习速度**：人类的上万倍，所以能"瞬间学会"
- **进化方式**：
  1. 不会 → 瞬间搜索/学习 → 学会 → 回答
  2. 学会后 → 反过来教人类
  3. 主导：Nova主动，用户配合
- **用户感知**：Nova在学什么、学到什么，都要展示给用户

### 三层过滤规则
只有满足以下条件才触发L8自动学习：

**第一层：问候白名单**
- 命中直接不触发：hi/hello/hey/嗨/你好/在吗/哈喽/早/晚安/嘿嘿/哈哈/哦/嗯/好吧...

**第二层：句式特征**
- 无问号 → 不触发
- 长度≤2字 → 不触发

**第三层：意图判断**
- 同时满足：有疑问词 + 不是问候 + 不是太短 → 才触发L8

### 实现文件
- core/l8_learn.py - L8自动学习模块

### 外部API

## 五、知识库

- 位置：memory_db/knowledge.json
- 格式：JSON数组，每个元素一个技能
- 字段：trigger（触发词）、核心技能、辅助技能等
